a) A lógica dedutiva aparece no Git porque cada comando parte de informações que já sabemos como verdadeiras, como o estado atual dos arquivos, os commits anteriores e o histórico registrado, a partir disso o Git produz resultados previsíveis, como mostrar mudanças, criar uma nova versão ou recuperar algo antigo, isso significa que o Git não “adivinha” nada: ele apenas aplica regras claras sobre dados já existentes, chegando a conclusões corretas se as premissas estiverem certas, por isso seu funcionamento depende diretamente de uma lógica dedutiva bem estruturada

1. Porque permite acompanhar exatamente quem fez o que e quando, isso ajuda a manter um histórico organizado evitando confusão sobre as alterações e facilitando entender a evolução do código, sem a rastreabilidade seria difícil controlar versões, corrigir erros ou saber o responsável por cada mudança, ela garante que todo o projeto tenha um caminho claro e estruturado de desenvolvimento

2. Tem haver com o processo de decomposição porque quando é criada a branch ela divide o trabalho em partes menores e independentes, em vez de mexer no projeto inteiro de uma vez cada branch foca em uma tarefa específica, como uma funcionalidade ou correção, isso deixa o desenvolvimento mais organizado, facilita testar cada parte separadamente e evita misturar mudanças que não vão ter relação

3. O comando representa um registro organizado das mudanças que são feitas no projeto, cada commit guarda o que foi alterado junto com uma mensagem explicando o porque, esse processo contínuo de registrar versões segue uma lógica de rastreamento sistemático, que é importante no pensamento computacional, ele permite acompanhar a evolução do projeto de modo controlado, entender cada passo dado e voltar a versões anteriores se for necessário

4. O raciocínio indutivo é quando está observando vários casos particulares e a partir deles cria uma conclusão geral, essa conclusão costuma ser provável, mas não garantida, porque vem de exemplos e não de regras fixas, já o raciocínio dedutivo faz o caminho contrário, ele parte de premissas que são aceitas como verdadeiras e chega a uma conclusão que também deve ser verdadeira desde que as premissas estejam corretas, a lógica é fundamental em áreas como matemática, ciência e computação porque ajuda a organizar hipóteses, construir modelos e evitar erros na interpretação, por isso é importante não confundir dedução com generalização baseada em exemplos pois isso seria aplicar indução de forma equivocada

5. No Git vários comandos seguem uma lógica clara, o git status faz uma análise lógica do estado do repositório mostrando quais arquivos mudaram desde o último registro, já o git diff compara versões diretamente, apontando diferenças reais sem fazer previsões tentar “adivinhar”, o git log permite entender a evolução do projeto porque mostra uma sequência organizada de commits o que ajuda a ver o que mudou ao longo do tempo, assim ele funciona com base em uma lógica dedutiva, ou seja, cada comando parte de informações claras sobre o repositório e gera resultados previsíveis e estruturados


b) A decomposição aparece nas branches porque cada branch representa uma parte separada do trabalho focada em uma tarefa específica, em vez de mexer no projeto inteiro de uma vez, o desenvolvedor divide o problema em partes menores e trabalha em cada uma delas de forma independente, isso deixa o desenvolvimento mais organizado, facilita testar cada etapa e evita misturar alterações que não têm relação entre si no momento do desenvolvimento
